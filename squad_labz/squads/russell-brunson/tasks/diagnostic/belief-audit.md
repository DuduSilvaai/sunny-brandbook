# Task: Belief Audit

> **Owner:** @russell-brunson
> **Frameworks:** Expert Secrets — The 3 Core Beliefs (Vehicle / Internal / External)
> **Inputs:** avatar completo (`*dream-customer-avatar`), produto/método, funil ou copy atual (se existir)

## Quando Executar

Antes de construir qualquer copy, webinar ou oferta. O Belief Audit garante que os 3 tipos de crença falsa do avatar sejam identificados e que o funil os endereça sequencialmente. Executar também como diagnóstico quando a copy existe mas a conversão está abaixo do threshold.

## Algoritmo

### Os 3 Tipos de Crença de Russell Brunson

Toda resistência à compra vem de uma das 3 crenças falsas abaixo. A copy deve quebrá-las nessa ordem específica — tentar fechar sem quebrar as 3 resulta em objeções no momento da venda.

**Crença 1 — Vehicle Belief (Crença no Veículo)**
"Esse método / produto / abordagem não vai funcionar para mim."
O avatar duvida do veículo (o método que você ensina), não de si mesmo.

**Crença 2 — Internal Belief (Crença Interna)**
"Mesmo que o método funcione para outros, EU não sou capaz de executar."
O avatar acredita no veículo mas duvida das próprias capacidades, disciplina ou talento.

**Crença 3 — External Belief (Crença Externa)**
"Mesmo que eu seja capaz, fatores fora do meu controle vão me impedir."
O avatar acredita no veículo e em si mesmo, mas vê obstáculos externos como intransponíveis (mercado, tempo, dinheiro, família, etc.).

### Passo 1: Identificar a Crença Falsa Exata (em palavras do avatar)

Para cada uma das 3 crenças:

**Formato de output:**
```
Crença [1/2/3]: [nome do tipo]
Crença falsa exata: "[frase em primeira pessoa, como o avatar pensaria]"
Fonte: onde coletou essa evidência (comentários, depoimentos, objeções de vendas, etc.)
Intensidade: [Alta / Média / Baixa] — quão difundida é essa crença no avatar médio
```

Exemplos por tipo:

Vehicle Belief falsa: "Já tentei [categoria de produto] antes e nunca funcionou. É sempre a mesma coisa com nome diferente."

Internal Belief falsa: "Eu entendo a teoria mas na hora de executar eu sempre trava. Não tenho disciplina suficiente."

External Belief falsa: "No meu mercado é diferente — está muito saturado / meu nicho é pequeno demais / não tenho budget para tráfego."

### Passo 2: Identificar a História que Criou a Crença

Para cada crença falsa mapeada:
- Que experiência anterior do avatar solidificou essa crença?
- Que figura de autoridade reforçou essa crença?
- Qual evento negativo traumático relacionado a tentativas anteriores?

**Por que isso importa:** A nova história que você conta no funil deve ser do mesmo "tipo" emocional da história negativa do avatar, mas com desfecho oposto.

### Passo 3: Definir a Nova Crença a Instalar

Para cada crença falsa, definir a crença oposta que, se instalada, torna a objeção inativa:

| Crença Falsa | Nova Crença |
|-------------|------------|
| "[Vehicle]: método não funciona" | "[Vehicle]: esse método específico é diferente porque [razão verificável]" |
| "[Internal]: eu não sou capaz" | "[Internal]: qualquer pessoa que [característica que o avatar tem] consegue executar" |
| "[External]: fatores externos me impedirão" | "[External]: o obstáculo [X] é na verdade [reframe que o remove]" |

### Passo 4: Identificar a História ou Framework que Instala a Nova Crença

Para cada nova crença, identificar:
- Qual história do expert ou de cliente instala essa crença? (Epiphany Bridge específica para aquela crença)
- Qual framework / metodologia demonstra que a crença é verdadeira?
- Qual prova social (depoimento / resultado) confirma a nova crença?

### Passo 5: Mapear Onde no Funil Cada Crença é Endereçada

| Crença | Onde endereçar | Mecanismo |
|--------|---------------|-----------|
| Vehicle | Secret #1 + Step 7 (webinar) | Epiphany Bridge do expert |
| Internal | Secret #2 + Step 9 (webinar) | Caso de alguém similar ao avatar |
| External | Secret #3 + Step 11 (webinar) | Reframe + prova que obstáculo é superável |

Se o funil não for webinar, mapear para o equivalente na estrutura escolhida (VSL, email, etc.).

### Verificação de Completude

Checklist de validação antes de considerar o Belief Audit completo:

- [ ] As 3 crenças falsas foram identificadas em palavras do avatar (não paráfrase do expert)
- [ ] A história que criou cada crença foi documentada
- [ ] A nova crença a instalar está formulada como afirmação (não negação da falsa)
- [ ] Existe história / prova / framework para instalar cada nova crença
- [ ] Cada crença está mapeada para um momento específico no funil
- [ ] Sequência respeita a ordem Vehicle → Internal → External

## Output

- Documento de Belief Audit com as 3 crenças completas (5 campos cada)
- Mapa de cobertura no funil (onde cada crença é quebrada)
- Lacunas identificadas (crenças sem cobertura adequada = risco de objeção na venda)
- Recomendação: quais histórias / provas criar para preencher as lacunas

## Thresholds

| Critério | Meta |
|---------|------|
| Crenças falsas identificadas | 3/3 obrigatório |
| Cada crença em palavras do avatar | Sim (não paráfrase genérica) |
| Cobertura no funil | 3/3 crenças com ponto de endereçamento definido |
| Prova ou história para cada nova crença | ≥ 1 por crença |
| Objeções de vendas cobertas pelas 3 crenças | ≥ 80% das objeções conhecidas |
