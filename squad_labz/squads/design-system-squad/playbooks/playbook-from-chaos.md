# Playbook: Consolidate & Audit Existing UI

For projects with **existing UI that needs consolidation**.

**Phases:** 0 → 1A → 1B → 2 → 3 → 4 → 5 → 6

## Phase 0: Business Discovery
**Agent:** @design-architect
**Duration:** 1-2 days
**Output:** `business-context.md`

## Phase 1A: Audit Existing UI
**Agent:** @brad-frost
**Duration:** 5-7 days
**Output:** Pattern redundancy analysis

- Scan entire codebase for patterns
- Calculate consolidation potential
- Identify orphan components
- Shock report for stakeholders

## Phase 1B: Establish Foundation
**Agents:** @alla-kholmatova + @brad-frost
**Duration:** 3-5 days
**Output:** Shared language + Consolidation architecture

## Phase 2: Consolidation Strategy
**Agents:** @dan-mall + @heydon-pickering
**Duration:** 5-7 days
**Output:** ROI case + Accessibility remediation plan

## Phase 3: Token Extraction & Migration
**Agents:** @jina-anne + @una-kravets
**Duration:** 7-10 days
**Output:** Token system + CSS migration path

- Extract tokens from consolidated patterns
- Plan CSS refactor
- Create migration guide

## Phase 4: Governance & Rollout Planning
**Agents:** @nathan-curtis + @dave-malouf
**Duration:** 3-5 days
**Output:** Governance model + Adoption strategy

## Phase 5: Accessibility Validation
**Agent:** @heydon-pickering
**Duration:** 3-5 days
**Output:** Accessibility audit + Remediation plan

## Phase 6: Phased Migration
**Agent:** @design-architect (orchestrates)
**Duration:** Ongoing (4-8 weeks)
**Output:** Teams migrated + Old patterns deprecated

---

**Total Duration:** ~40-50 days of focused work
**Team Size:** 3-4 people
**Success Criteria:** 70%+ pattern consolidation, 95%+ token adoption, WCAG 2.2 AA minimum
