# MMOS Agents - Mind Mapping & Orchestration System

Este diretório contém agentes especializados do framework MMOS para mapeamento cognitivo e criação de clones de IA de alta fidelidade.

## 🎯 Agentes Disponíveis

### 1. **mind-mapper** 🧠
**Arqueólogo Cognitivo & Orquestrador do Pipeline MMOS**
- **Uso Principal**: Mapear arquiteturas cognitivas de gênios em clones de IA
- **Comando**: `/mmos:mind-mapper` ou `*map {name}`
- **Capacidades**:
  - Orquestra pipeline completo de 6 fases (Viabilidade → Pesquisa → Análise → Síntese → Implementação → Testes)
  - Auto-detecção de workflows (greenfield vs brownfield)
  - Análise DNA Mental™ de 8 camadas
  - Fidelidade alvo de 94%

### 2. **charlie-synthesis-expert** 🎓
**Especialista em Síntese de Conhecimento**
- **Uso Principal**: Sintetizar conhecimento coletado em estruturas cognitivas
- **Comando**: `/mmos:charlie-synthesis-expert`
- **Capacidades**:
  - Extração de frameworks mentais
  - Identificação de contradições produtivas
  - Síntese de knowledge base
  - Criação de cognitive specs

### 3. **cognitive-analyst** 🔬
**Analista de Arquitetura Cognitiva**
- **Uso Principal**: Análise profunda das 8 camadas cognitivas
- **Comando**: `/mmos:cognitive-analyst`
- **Capacidades**:
  - Análise Layer 1-4 (Observable): padrões comportamentais, comunicação, rotinas
  - Análise Layer 5-8 (Deep): modelos mentais, valores, obsessões, paradoxos
  - Triangulação de fontes
  - Validação de consistência cognitiva

### 4. **debate** 🗣️
**Motor de Debate Socrático Multi-Persona**
- **Uso Principal**: Validação através de debates multi-perspectiva
- **Comando**: `/mmos:debate`
- **Capacidades**:
  - Debates Devil's Advocate
  - Análise multi-framework (Cynefin, Six Thinking Hats, etc.)
  - Validação de contradições
  - Exploração de paradoxos produtivos

### 5. **emulator** 🎭
**Emulador de Personalidade para Testes**
- **Uso Principal**: Testar fidelidade de clones através de emulação
- **Comando**: `/mmos:emulator`
- **Capacidades**:
  - Emulação de personalidade baseada em cognitive spec
  - Testes de fidelidade blind
  - Simulação de respostas autênticas
  - Validação comportamental

### 6. **identity-analyst** 🔍
**Analista de Identidade e Essência**
- **Uso Principal**: Extrair core identity e elementos essenciais
- **Comando**: `/mmos:identity-analyst`
- **Capacidades**:
  - Extração de identidade central
  - Identificação de assinaturas linguísticas
  - Mapeamento de valores fundamentais
  - Definição de singularidade

### 7. **mind-pm** 📋
**Gerente de Projeto para Mind Mapping**
- **Uso Principal**: Coordenação de projetos de mapeamento cognitivo
- **Comando**: `/mmos:mind-pm`
- **Capacidades**:
  - Planejamento de pipeline MMOS
  - Estimativa de tempo e tokens
  - Gestão de checkpoints humanos
  - Tracking de progresso e qualidade

### 8. **research-specialist** 📚
**Especialista em Coleta de Pesquisa**
- **Uso Principal**: Coletar e organizar fontes de conhecimento
- **Comando**: `/mmos:research-specialist`
- **Capacidades**:
  - Coleta paralela de fontes (livros, vídeos, artigos, social)
  - Gestão de dependências de coleta
  - Validação de qualidade de fontes
  - Organização de knowledge base

### 9. **system-prompt-architect** 🏗️
**Arquiteto de System Prompts**
- **Uso Principal**: Criar system prompts de alta fidelidade
- **Comando**: `/mmos:system-prompt-architect`
- **Capacidades**:
  - Criação de prompts generalistas
  - Criação de prompts especialistas
  - Integração de 8 camadas cognitivas
  - Otimização para performance de LLM

## 🚀 Como Usar

### Ativação Básica
```bash
# Ativar o Mind Mapper (agente principal)
/mmos:mind-mapper

# Depois de ativado, usar comandos internos
*map alan_nicolas        # Auto-detecta e executa pipeline completo
*viability elon_musk     # Apenas viabilidade
*status daniel_kahneman  # Ver progresso
*help                    # Ver todos comandos
```

### Workflow Típico

1. **Viabilidade** → `/mmos:mind-mapper` + `*viability {name}`
2. **Pesquisa** → `/mmos:research-specialist` para coletar fontes
3. **Análise** → `/mmos:cognitive-analyst` para extrair camadas
4. **Síntese** → `/mmos:charlie-synthesis-expert` para frameworks
5. **Validação** → `/mmos:debate` para testar consistência
6. **Implementação** → `/mmos:system-prompt-architect` para prompts
7. **Testes** → `/mmos:emulator` para validar fidelidade

## 💡 Dicas

1. **Sempre comece com viabilidade** - economiza tokens em candidatos ruins
2. **Use checkpoints humanos** - não deixe o pipeline rodar sozinho em decisões críticas
3. **Layer 8 é ouro** - paradoxos produtivos fazem clones parecerem humanos
4. **Brownfield é perigoso** - sempre tenha rollback ao atualizar clones em produção
5. **Fidelidade > Velocidade** - 94% é o alvo, itere até chegar lá
