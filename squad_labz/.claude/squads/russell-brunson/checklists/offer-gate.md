# Offer Gate

> **Aprovacao:** @russell-brunson
> **Regra:** 100% dos itens com [x] antes de usar a oferta em qualquer funil ou webinar
> **Referencia:** `tasks/generative/offer-stack-builder.md`

---

## Stack Slide — Estrutura

- [ ] Produto core definido com nome comercial e transformacao central em 1 frase
- [ ] Stack com minimo 5 itens (core + minimo 4 bonus)
- [ ] Cada item do Stack tem nome proprietario (nao "Bonus 1", "Bonus 2")
- [ ] Cada item tem utilidade autonoma (vale mesmo sem o produto core)
- [ ] Sequencia de revelacao definida: do maior para o menor valor percebido

---

## Valor Percebido

- [ ] Valor percebido definido para cada item (benchmark de mercado documentado)
- [ ] Total do Stack calculado (soma de todos os valores percebidos)
- [ ] Relacao valor percebido / preco cobrado >= 10x
- [ ] Justificativa do valor de cada item e verificavel pelo avatar

---

## Cobertura de Crencas e Objecoes

- [ ] Vehicle Belief (metodo nao funciona): endereçada por historia, framework ou bonus especifico
- [ ] Internal Belief (eu nao sou capaz): endereçada por caso de cliente similar ao avatar ou bonus de implementacao
- [ ] External Belief (fatores externos me impedirao): endereçada por bonus ou reframe na copy
- [ ] As 3 objecoes mais comuns do avatar cobertas por algum item do Stack

---

## Garantia e Urgencia

- [ ] Garantia configurada: tipo (satisfacao / resultado) + prazo (minimo 30 dias) + processo de reembolso claro
- [ ] Garantia apresentada como inversao de risco (nao como condicao restritiva)
- [ ] Urgencia real definida — razao legitima documentada (vagas, prazo, bonus com data)
- [ ] Urgencia nao e artificial (nao usar "quantidade limitada" sem limite real)

---

## OTO (Order Bump / Upsell)

- [ ] OTO 1 complementa o core (nao repete o que o core ja entrega)
- [ ] OTO 1 tem preco menor que o core (regra: OTO <= 50% do preco do core)
- [ ] Downsell definido caso avatar recuse OTO 1
- [ ] Sequencia de OTO nao tem mais de 2 etapas antes da pagina de obrigado real

---

## Script de Revelacao

- [ ] Script do Stack Slide escrito (revelacao item por item com valor acumulado)
- [ ] Frase de encerramento definida ("O valor total e R$X, mas hoje...")
- [ ] Justificativa do preco menor e legitima e esta no script
- [ ] CTA unico apos revelacao do preco (sem alternativas que dispersem atencao)
