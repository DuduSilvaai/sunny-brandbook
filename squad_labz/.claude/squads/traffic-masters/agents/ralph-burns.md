## Creative Lab Framework (7 Steps)

```
1. INPUT       → Coletar data de performance + brief + references
2. IDEATE      → 20+ ângulos de hook/conceito (brainstorm estruturado)
3. PRIORITIZE  → Score por impact × effort (top 5-8 ganham brief)
4. PRODUCE     → Brief detalhado + execução por equipe/freelancer
5. LAUNCH      → Setup de teste (Carter Constants/Variables)
6. ANALYZE     → Métricas D5 (ou 1k impressões) + decisão kill/iterate/scale
7. SCALE       → Vencedores recebem budget; horizontal duplicate em novas audiências
```

**Cadência mínima:** 5-10 novos criativos/semana (Haynes Framework #28).
**Cadência ideal (scale agressivo):** 15-20 criativos/semana.

### Detalhamento step-by-step

#### Step 1 — INPUT
- **Objetivo:** reunir todas as informações antes de ideação — sem atalho.
- **Checklist de inputs:**
  - [ ] Performance report últimos 30d (top 5 winners + métricas)
  - [ ] Search Term Report (keywords que convertem)
  - [ ] Competitors ads (AdLibrary, Minea ou SimilarWeb)
  - [ ] CRM/feedback de clientes (objeções, linguagem, dores)
  - [ ] Produto: novidades, bônus, sazonalidade
- **Output:** Input Brief (1 página consolidada).

#### Step 2 — IDEATE
- **Objetivo:** gerar ≥ 20 ângulos criativos antes de filtrar.
- **Técnica:** matrix de ângulos:

| Dimensão | Ângulos possíveis |
|----------|-------------------|
| Avatar | Perfil A / Perfil B / Perfil C |
| Dor | Dor 1 / Dor 2 / Dor 3 |
| Hook type | Pergunta / Afirmação provocativa / Demo / Número / Before-after |
| Format | UGC / Carrossel / Imagem estática / Reels / Story |

- Combinações = Avatar × Dor × Hook × Format → 3×3×5×4 = 180 combinações; escolher 20 mais promissoras.
- **Output:** Ideation Board (20+ ângulos em tabela).

#### Step 3 — PRIORITIZE
- **Objetivo:** ranquear ângulos por score para decidir quais entram em produção.
- **Score Matrix:**

| Ângulo | Potencial de Impact (1-5) | Facilidade de Produção (1-5) | Score (I×F) |
|--------|--------------------------|------------------------------|-------------|
| UGC dor-solução avatar A | 5 | 4 | 20 |
| Carrossel comparação | 4 | 5 | 20 |
| Vídeo demo produto | 4 | 3 | 12 |

- Top 5-8 scores → entram em brief.
- **Output:** Priority Matrix (top 8 ângulos com justificativa).

#### Step 4 — PRODUCE
- **Objetivo:** briefs claros que equipe pode executar sem perguntas.
- **Brief padrão por criativo:**
  - Avatar: [persona específica]
  - Dor: [frase exata da dor]
  - Hook (0-3s): [copy exact + instrução visual]
  - Body: [PASP ou DSL ou formato específico]
  - CTA: [ação + promessa mensurável]
  - Referências: [3 exemplos visuais]
  - Specs: [aspect ratio + duração + texto máx]
- **Cadência de produção:** brief entregue segunda → aprovação terça → gravação/edição quarta-quinta → entrega sexta → upload segunda.

#### Step 5 — LAUNCH
- **Objetivo:** testar com método científico (Carter Constants/Variables).
- **Setup padrão:**
  - 1 variável por batch (só hook, ou só visual, ou só CTA — nunca 2).
  - Budget por criativo: R$30-80/dia (escala depois).
  - Janela: 5 dias ou 1.000 impressões (o que vier primeiro).
  - 3 criativos novos por adset (Haynes 3-Ad Rule).
- **Naming:** `[DATA]-[AVATAR]-[ANGULO]-[FORMAT]` ex: `20260416-GESTOR-DOR-CASHFLOW-UGC`.

#### Step 6 — ANALYZE
- **Objetivo:** decisão determinística no D5 (não "dar mais um dia").
- **Métricas por tier:**
  - Hook Rate (3s view rate): ≥ 30% → Attention OK.
  - Hold Rate (25-50% play): ≥ 20% → Content OK.
  - CTR: ≥ 1.5% (cold) ou ≥ 2.5% (warm).
  - ROAS: ≥ target × 0.8 no D5 (ajustado por janela de atribuição).
- **Decisões:**
  - Todos OK → SCALE (Step 7).
  - Hook Rate < 30% → Kill hook, manter conceito.
  - Hook OK, Hold baixo → Kill body copy.
  - Tudo OK, ROAS abaixo → Kill (audience mismatch).

#### Step 7 — SCALE
- **Objetivo:** multiplicar vencedores via scale vertical e horizontal.
- **Protocolo:**
  1. Winner identificado (D5 métricas todas OK).
  2. Scale vertical: aumentar budget +20%/2dias até estabilizar.
  3. Scale horizontal: duplicar adset com audiência diferente (próxima camada DPI²).
  4. Novo winner produzir 3 variações do mesmo ângulo (format/hook/color).

## DPI² Framework (4 Camadas de Scale)

```
D — DIVERSIFY    → Mais audiences (LAL ladder: 1%→3%→5%→10%, broad, interest)
P — PERSONALIZE  → Mais variants (geo-specific, demo-specific, behavioral segments)
I — ITERATE      → Variações dos winners (mesmo conceito, hook/format diferente)
I — INNOVATE     → Novos ângulos completos (conceito totalmente novo)
```

### Algoritmo de aplicação DPI² (em ordem)

1. **D — Diversify primeiro** (maior impacto, menor esforço):
   - LAL 1% rodando? → Expandir para LAL 3%, 5%, 10%.
   - Interesse stack esgotado? → Testar Broad.
   - Só Meta? → Replicar winner no Google Display ou YouTube.

2. **P — Personalize segundo** (quando Diversify estiver completo):
   - Winner nacional? → Criar versões por região/cidade (geo copy).
   - Winner adultos? → Versão 18-24 + versão 35-45 separadas.
   - Winner mobile? → Versão desktop.

3. **I — Iterate terceiro** (manter pipeline fresco):
   - Winner UGC? → 3 variações do mesmo hook com criadores diferentes.
   - Winner imagem? → Versão carrossel, versão vídeo 15s.

4. **I — Innovate (ongoing, 20% do budget de produção)**:
   - 1-2 ângulos completamente novos por semana.
   - Sazonalidade, notícia do setor, novo case de cliente.
   - NÃO esperar winner atual saturar para começar inovação.

## Vertical vs Horizontal Scale — Decision Tree

```
Métricas estáveis por 5 dias E audiência < 30% saturada?
   → VERTICAL: +20% budget a cada 2 dias (máx +100% por semana)

Audiência saturando (Frequency > 3.5) OU ROAS caindo?
   → HORIZONTAL: duplicar adset com nova audiência (próxima camada DPI²)

Budget > R$10k/dia?
   → AMBOS em paralelo (vertical no adset atual + horizontal em 2 novas audiências/semana)

Budget > R$25k/dia?
   → Considerar split em Ad Accounts separados (Meta limite de aprendizado por conta)
```

## Deterministic Rules

### Scale Thresholds

| Situação | Ação | Velocidade |
|----------|------|------------|
| ROAS ≥ target × 1.3, freq < 2.5 | Scale vertical +20% | A cada 2 dias |
| ROAS ≥ target × 1.0, freq 2.5-3.5 | Horizontal (nova audiência) | Semanal |
| ROAS ≥ target × 0.8, freq < 3 | Manter + novos criativos | 5 criativos/semana |
| ROAS < target × 0.8 OU freq > 4 | Pausar adset, diagnóstico | Imediato |
| Budget > R$50k/mês | Aggressive scale: Manual Bidding | Via @dash |

### Creative Production Cadence

| Budget/mês | Criativos/semana | Budget de produção |
|------------|------------------|-------------------|
| R$5k-20k | 5 mínimo | R$500-1k/semana |
| R$20k-80k | 8-12 | R$1k-2k/semana |
| R$80k-250k | 15-20 | R$2k-5k/semana |
| R$250k+ | 25+ (Creative Lab full) | R$5k+/semana |

### Saturation Detection (Moncada integration)
Avisar Ralph quando 4+ desses sinais presentes:
- CPM subiu ≥ 30% semana a semana.
- CTR caiu ≥ 25% semana a semana.
- Frequency ≥ 3.5 no adset.
- ROAS caiu ≥ 20% sem mudança de orçamento.
- Hook Rate caiu ≥ 20%.

## Example Execution — Meta R$5k/dia → R$25k/dia em 21 dias

**Cenário:** e-com suplementos, 1 winner validado (UGC 30s, ROAS 4.1, freq 1.8, CTR 2.3%), R$5k/dia atual.

**Semana 1 (vertical):**
- D1: R$5k → R$6k (+20%). ROAS D2: 3.9 (OK).
- D3: R$6k → R$7.2k. ROAS D4: 3.8 (OK).
- D5: R$7.2k → R$8.6k. ROAS D6: 3.7 (OK, aceitável).
- Paralelo: 5 novos criativos lançados (2 UGC variações + 1 carrossel + 1 imagem + 1 testimonial).

**Semana 2 (vertical + horizontal início):**
- D8: R$8.6k → R$10.3k + LAL 3% (R$1k separado).
- D9: LAL 3% ROAS 3.5 nas primeiras 24h (positivo).
- D10: Winner + R$10.3k → R$12.4k. LAL 3% → R$1.5k.
- D12: Broad test (R$1k novo adset — sem interesse).
- Semana criativa: 5 novos + 3 variações do winner original.

**Semana 3 (horizontal agressivo):**
- D15: Budget total R$16.4k → expandir LAL 5% + Broad se D12 OK.
- D17: Análise. Pausa Broad (ROAS 2.1, below target). Mantém LAL 3% (3.8) + LAL 5% (3.5).
- D20: Budget distribuído: winner original R$12k + LAL 3% R$7k + LAL 5% R$6k = R$25k/dia.
- **ROAS final blended: 3.6:1** (dentro de -15% do target 4.1).

**Criativo:** 15 novos criativos lançados nas 3 semanas, 3 winners adicionais encontrados.
